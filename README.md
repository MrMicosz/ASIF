# ASIF: Adaptive Soft Identity Framework
**Adaptive Soft Identity Framework for Cybersecurity Risk Detection in Digital Financial Systems**

---

## ที่มาและแนวคิดของโครงการ
ในปัจจุบัน สังคมดิจิทัลพึ่งพาระบบออนไลน์และธุรกรรมดิจิทัลเพิ่มขึ้นอย่างต่อเนื่องอย่างไรก็ตาม ระบบยืนยันตัวตนกระแสหลักที่เรียกว่า **Hard Identity** (เช่น เลขบัตรประชาชน, ข้อมูลชีวมาตร, รหัสผ่าน หรือ OTP) เริ่มแสดงให้เห็นถึงข้อจำกัด
* **ช่องโหว่ทางสังคม:** หากอาชญากรสามารถครอบครองข้อมูลเหล่านี้ได้ผ่านการหลอกลวง (Social Engineering) ระบบจะไม่สามารถแยกแยะได้ว่าผู้ใช้งานคือเจ้าของตัวจริงหรือผู้ไม่หวังดี
* **ปัญหาบัญชีม้า:** บัญชีที่เปิดถูกต้องตามกฎหมายทุกประการ แต่ถูกนำไปใช้ในลักษณะผิดปกติ ซึ่งระบบ Hard Identity เพียงอย่างเดียวตรวจจับได้ยาก

**ASIF** จึงถูกพัฒนาขึ้นภายใต้แนวคิด **Soft Identity** ซึ่งเป็นตัวตนแบบยืดหยุ่นที่สะสมมาจากพฤติกรรมการใช้งานในหลายมิติเพื่อสร้างกลไกการยืนยันและประเมินความเสี่ยงที่ก้าวไปไกลกว่าการตรวจสอบเอกสารหรือรหัสผ่านแบบเดิม

### จุดเด่นและนวัตกรรม
* **Cognitive Friction:** การใช้ AI แจ้งเตือนในลักษณะ Chat Head เพื่อสร้างแรงต้านทางความคิด กระตุ้นให้ผู้ใช้งานหยุดทบทวนการตัดสินใจในสถาณการณ์ที่มีความเสี่ยงสูง
* **AI Support Chatbot:** ระบบสนับสนุนที่ช่วยวิเคราะห์ข้อมูลต้องสงสัย เช่น ลิงก์ หรือบริบทการสนทนา เพื่อให้ความรู้และลดความตื่นตระหนกของผู้ใช้
* **Behavioral Network Analysis:** สำหรับกรณีบัญชีใหม่ที่ยังไม่มีข้อมูลเดิม ระบบจะใช้ AI เรียนรู้รูปแบบจากเครือข่ายบัญชีม้าในอดีต (Scammer Network) เพื่อเฝ้าระวังความเสี่ยงในอนาคต

---

## กลไกการทำงาน
ระบบจะใช้ Machine Learning / AI ในการเรียนรู้พฤติกรรมเพื่อสร้าง **Baseline Soft Identity**:
1. **Data Dimensions:** รวบรวมข้อมูลทั้งรูปแบบการพิมพ์ (Typing Patterns), การสัมผัสหน้าจอ (Touch Dynamics), อุปกรณ์, ช่วงเวลา, และลำดับการทำธุรกรรม
2. **Deviation Assessment:** เมื่อเกิดธุรกรรมใหม่ AI จะเปรียบเทียบกับ Baseline เพื่อประเมินระดับความเบี่ยงเบนของพฤติกรรม
3. **Risk Scoring:** แปลงผลลัพธ์เป็นคะแนนความเสี่ยง (Risk Score 0-100%) เพื่อเป็นข้อมูลประกอบการตัดสินใจของสถาบันการเงิน

## การอธิบายแนวคิดผ่านภาพ
เพื่อให้เห็นภาพการทำงานของระบบและชั้นข้อมูลของ Soft Identity ได้ชัดเจนยิ่งขึ้น ท่านสามารถเลือกดูแผนภาพ Visualization ได้ 2 ช่องทาง:

1.  **Local HTML:** เปิดไฟล์ `Visualization.html` ในโฟลเดอร์โครงการเพื่อดูแผนภาพแบบ Interactive
2.  **Online Dashboard:** เข้าชมผ่าน [Hugging Face Spaces: ASIF](https://huggingface.co/spaces/Micosz/ASIF)
    * *หมายเหตุ: หากลิงก์ Hugging Face มีปัญหาทางเทคนิคหรือโหลดช้า แนะนำให้เปิดผ่านไฟล์ HTML ในเครื่องแทน*

## ขั้นตอนการทดลองใช้งาน Demo
*หมายเหตุ: ส่วนนี้เป็นชุดโปรแกรมสำหรับจำลองการเรียนรู้พฤติกรรมเบื้องต้นเพื่อแสดงความเป็นไปได้ของแนวคิดเท่านั้น*

### 1. การสร้างข้อมูลสำหรับ Train
เข้าใช้งานไฟล์ **`SelfTrainDataApplication`** เพื่อสร้างข้อมูลพฤติกรรมเริ่มต้น โดยการพิมพ์ข้อความในกรอบที่กำหนดและกด **Save** เพื่อบันทึกเป็นไฟล์ **JSON**

### 2. การเทรนโมเดล
เข้าใช้งานไฟล์ **`MainEngineForStd-Train`** เพื่อให้ AI เรียนรู้รูปแบบพฤติกรรมจากไฟล์ JSON และสร้างโมเดลอัตลักษณ์ (**`.pkl`**)

### 3. การสร้างข้อมูลสำหรับตรวจสอบ
กลับไปที่ **`SelfTrainDataApplication`** อีกครั้งเพื่อสร้างข้อมูลชุดใหม่สำหรับการทดสอบ (Test Set) ทั้งในกรณีที่เป็นเจ้าของบัญชีพิมพ์เอง หรือจำลองให้ผู้อื่นมาพิมพ์แทน

### 4. การตรวจสอบผลลัพธ์
เข้าใช้งานไฟล์ **`Verify`** เพื่อเปรียบเทียบข้อมูลใหม่กับโมเดลที่สร้างไว้ ระบบจะแสดงผลลัพธ์การยืนยันตัวตนเชิงพฤติกรรม
