Adaptive Soft Identity Framework

ยกระดับความปลอดภัยระบบการเงินดิจิทัลด้วย "อัตลักษณ์เชิงพฤติกรรม"
นวัตกรรมที่ปรับตัวตามตัวตนจริงของผู้ใช้งานในทุกวินาที

เริ่มต้นสำรวจ ลองทดสอบ AI

ทำไม "Hard Identity" ถึงไม่เพียงพอ?

ในยุคที่มิจฉาชีพหันมาใช้ Social Engineering มากขึ้น ระบบความปลอดภัยที่พึ่งพาเพียง Hard Identity (รหัสผ่าน, OTP, ใบหน้า) ไม่สามารถป้องกันภัยที่เกิดจากการโน้มน้าวใจได้ เพราะผู้ใช้งานเป็นคนกดดำเนินการด้วยตัวเอง

The Missing Link:

  • บัญชีม้าที่ผ่าน KYC อย่างถูกต้อง แต่พฤติกรรมเปลี่ยนไป
  • ความลังเลระหว่างพิมพ์ (Hesitation) ที่บ่งบอกถึงภาวะถูกบีบคั้น
  • ความปลอดภัยที่จบลงทันทีหลังจากขั้นตอนการ Login

ASIF: เติมเต็มช่องว่าง

เราเปลี่ยนจากการถามว่า "คุณมีอะไร?" (What you have)
เป็นการสังเกตและเรียนรู้ว่า "คุณทำอย่างไร?" (How you do) เพื่อระบุตัวตนแบบต่อเนื่องตลอดเซสชัน

อัตลักษณ์เชิงพฤติกรรม (Soft Identity)

3 มิติข้อมูลพฤติกรรมที่ ASIF ใช้สร้าง "ลายนิ้วมือดิจิทัล" ของคุณ

Keystroke

วิเคราะห์จังหวะการพิมพ์ ความเร็ว และแรงกดของนิ้ว ซึ่งมีรูปแบบเฉพาะตัวที่ปลอมแปลงได้ยากแม้จะรู้รหัสผ่าน

Touch Dynamics

ทิศทางการลากนิ้ว (Swipe Pattern) และพื้นที่สัมผัสบนหน้าจอที่สะท้อนถึงความถนัดทางกายภาพของแต่ละคน

Context Data

มิติของสถานที่ (GPS), ช่วงเวลาที่ใช้งาน และอุปกรณ์ประจำตัว (Hardware Profile) มาคำนวณร่วมกันเพื่อประเมินความเสี่ยง

AI Core Simulation: One-Class SVM

จำลองหัวใจของ ASIF ในการสร้างขอบเขตพฤติกรรมเพื่อตรวจจับความผิดปกติ

Cognitive Friction

สร้าง "แรงเสียดทานเชิงบวก"

เมื่อ AI ตรวจจับความผิดปกติ ASIF จะไม่บล็อกทันที แต่จะสร้าง Chat Head เพื่อกระตุ้นให้ผู้ใช้ "หยุดคิดและทบทวน" (Decelerate Decision) ซึ่งเป็นอาวุธที่ได้ผลที่สุดกับ Scammer

User-Centered Design

เราเน้นความโปร่งใส ผู้ใช้งานคือเจ้าของข้อมูล ASIF ทำงานอยู่เบื้องหลังเพื่อซัพพอร์ต ไม่ใช่เพื่อครอบงำหรือควบคุมสิทธิเสรีภาพของผู้ใช้

Intervention Strategy & SCD

นวัตกรรมการตรวจจับการถูกบังคับด้วยความเร็ว (Soon Confidence Detector)

Intervention Challenge

เมื่อความเชื่อมั่นต่ำ (Low Confidence) ระบบจะส่ง Challenge ที่ต้องใช้การตัดสินใจของมนุษย์จริง ไม่ว่าจะส่งไปหาผู้ใช้หรือส่งไปให้กับทางเจ้าของเพื่อดึงสติผู้ใช้

Adaptive Feedback Loop

Biological Signature Analysis

การวิเคราะห์มิติพฤติกรรมเชิงลึกที่ "บอทสุ่มค่า" ไม่สามารถเลียนแบบได้

Pattern Consistency vs Random Noise

แม้บอทจะถูกโปรแกรมให้สุ่มค่า (Random Noise) เพื่อหลบเลี่ยงการตรวจจับ Jitter แบบเดิม แต่สิ่งที่บอทขาดคือ "Muscle Memory Correlation" หรือความสัมพันธ์ของจังหวะที่เกิดจากความเคยชินของกล้ามเนื้อมนุษย์จริง

ASIF วิเคราะห์ 5 มิติสำคัญเพื่อแยกแยะลายนิ้วมือทางพฤติกรรมที่แท้จริงออกจากระบบอัตโนมัติที่พยายามปลอมตัว

ASIF Technical Architecture

การปกป้องแบบ Passive Mode ที่ทำงานเบื้องหลัง 24/7 (อ้างอิงจาก Pipeline ใน PDF)

Data Collector (SDK)

ฝังตัวเป็น SDK ในแอปธนาคาร ดักจับเหตุการณ์ KeyDown/KeyUp, TouchStart และ SensorData เข้ารหัสทันทีด้วย Public Key + TLS 1.3 เพื่อความปลอดภัยสูงสุด

Brain Processor (Server)

ระบบ Preprocessing หลังบ้านที่ทำการ Normalization ข้อมูลดิบเป็น Z-score และทำ Feature Fusion เพื่อสร้าง Unified Feature Vector (V_total)

Verifier (OCSVM)

โหลดโมเดลประจำตัวผู้ใช้และคำนวณ Decision Function เพื่อระบุผลลัพธ์แบบ Pass/Fail พร้อมปรับค่า Nu แบบ Dynamic ตามความเสี่ยงของบริบท

Privacy by Design &
Business Impact

ASIF ถูกออกแบบมาเพื่อธนาคารที่ใส่ใจทั้งความปลอดภัยและสิทธิส่วนบุคคล:

  • PDPA Compliant: ข้อมูลพฤติกรรมถูกเข้ารหัสและไม่สามารถย้อนกลับเป็นตัวตนจริงได้หากไม่มีกุญแจถอดรหัส
  • Zero Latency UX: การประมวลผลเบื้องหลังทำให้ผู้ใช้ไม่ต้องพิมพ์ OTP บ่อยครั้งในบริบทที่ปลอดภัย
  • Reduced Losses: ช่วยลดมูลค่าความเสียหายจาก Social Engineering และบัญชีม้าได้ถึง 70-80% ในระบบจำลอง

Future Roadmap

Phase 1

Alpha Testing

มุ่งเน้นการเก็บข้อมูลพฤติกรรมในกลุ่มปิดเพื่อสร้าง Baseline Model ที่มีความแม่นยำสูง (EER < 4%)

Phase 2

Network Integration

เชื่อมโยงฐานข้อมูลพฤติกรรมและความเสี่ยงข้ามสถาบันการเงิน เพื่อป้องกันบัญชีม้าในระดับเครือข่าย

Phase 3

Emotional Sensing

พัฒนาระบบให้ตรวจจับสภาวะทางอารมณ์ เช่น ความเครียดหรือความกลัว ผ่านรูปแบบการสัมผัสหน้าจอ

Phase 4

Global Standard

ยกระดับ ASIF สู่การเป็นมาตรฐานใหม่ของการยืนยันตัวตนในระดับสากล สำหรับทุกแพลตฟอร์มดิจิทัล
